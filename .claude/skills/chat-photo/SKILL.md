# chat-photo

把**一张（或多张）聊天截图**变成**卡通涂鸦风的人物对话漫画 PNG**，多格分镜，直接可发小红书。

风格基准 = 粗黑描边 + 扁平填色的圆润小动物（红兔 / 绿兔那类），配爱心、星星、闪光等手绘小装饰，纯色块 + 渐变背景。视觉规范沿用本 skill 自带的 `template.html`。

## 触发条件

当用户说以下任意一种时启动：
- `/chat-photo`
- "把这张聊天截图做成漫画"、"聊天记录生成对话漫画"
- "把我们的对话画成漫画"、"做成情侣 / 朋友 / 吵架漫画"
- "把聊天截图变成 XX 风格漫画"

## 核心思路（三段式）

**看图提取对话 → 规划角色与分镜 → HTML 排版渲染 PNG**

1. **Claude 直接看图**（`Read` 工具读截图）提取对话，无需 OCR；
2. 角色锁定在「粗黑描边 + 扁平填色 + 圆润可爱」的涂鸦风，默认小兔，可换其它动物；
3. 用 **CSS 矢量角色**（不依赖 AI 绘图）+ HTML/CSS 排对话气泡分镜，`shot-scraper` 渲染 PNG。

> **为什么用 CSS 矢量而非 AI 绘图**：本流程默认零依赖、零 API key、即时出图。若要更精细的 AI 手绘立绘，需用户自备文生图 API key（如火山方舟 Seedream / OpenAI gpt-image-1），届时把角色 `<img>` 换成生成的 PNG 即可，其余排版不变。

## 执行流程

### 第 1 步：看图提取对话

用 `Read` 逐张读取用户给的聊天截图，提取成结构化对话：

```json
{
  "speakers": ["对方", "自己"],
  "lines": [
    {"speaker": "自己",  "type": "text",  "text": "在吗"},
    {"speaker": "对方",  "type": "text",  "text": "在的，怎么啦"},
    {"speaker": "对方",  "type": "emoji", "text": "[偷笑]"}
  ]
}
```

提取要点：
- **气泡颜色定身份（首要规则）**：**绿色气泡 = 自己**（微信里自己发的，通常靠右）；**白色 / 灰色气泡 = 对方**（通常靠左）。颜色优先于左右位置——若颜色与位置冲突，以颜色为准。
- **语音转文字白框归语音发送方**：语音消息（绿色"X″"气泡）下方紧跟的白框是**微信语音转文字结果**，其文字内容**归这条语音的发送方**（即自己），不是对方说的。识别要点：白框紧贴在某条语音气泡下方、与之同侧成组。
- **只保留有内容的聊天文本**：文字消息 + 语音转文字内容 + emoji / 表情包（表情包转成情绪标注，如 `[狗头]`、`[捂脸]`）；
- **忽略与聊天内容无关的元信息**：语音时长（`14"`、`4"`）、时间戳（`00:20`）、"已读 / 撤回 / 拍一拍"、"X 发起了语音通话"等系统提示——这些**不进漫画**，也不要用括号补成台词（如不要写"（发了段语音）""（深夜 00:20）"）；
- 语音消息**没有转文字白框时直接跳过**（无可辨内容）；有转文字白框则取白框文字、归发送方；
- 图片 / 红包等若**承载剧情**，才用括号极简注明（`（发了张照片）`、`（转账 ¥520）`），否则也忽略；
- 多张截图按时间顺序拼成一条完整对话流（仅用时间排序，时间本身不出现在漫画里）。

### 第 2 步：规划角色 + 分镜（窗格数自动判断）

**角色（默认小兔，可换）**：
- 两个 speaker 映射成两只**同种、不同配色**的动物，全程保持一致，只换表情和动作；
- 默认：撒娇/主动的一方 → 红兔，理性/回应的一方 → 绿兔；
- 可换猫 / 柴犬 / 熊 / 企鹅等，但务必死守「粗黑描边 + 扁平填色 + 圆润可爱」。见下方【备选动物配色表】。

**窗格数 = 由对话信息量自动决定**：
- 极短对话（1-2 个回合）→ **4 格**；
- 中等（3-5 回合）→ **6 格**（2×3）；
- 较长（6+ 回合，有起承转合）→ 最多 **9 格**（3×3）；
- 一格 = 一个**对话回合**：一句关键台词 + 说话角色的对应表情 + 一个气泡；
- 合并零碎短句，挑**有戏剧张力 / 有梗 / 有反转**的回合，不逐字照搬；
- 末格尽量收一个**梗 / 金句 / 反转**。

### 第 3 步：选风格（4 种预设）

复制 `template.html`，把 `<body class="theme-XXX">` 换成对应风格类名。不确定就用 `AskUserQuestion` 问用户，或按对话氛围自动选：

| 风格类名 | 名称 | 背景 / 气质 | 适合主题 |
|---|---|---|---|
| `theme-candy` | 🍬 粉色糖果（默认）| 粉红渐变 + 粉色地面 | 情侣、闺蜜、撒糖 |
| `theme-fresh` | 🌿 冷调清新 | 蓝绿渐变 + 薄荷地面 | 朋友、日常、职场 |
| `theme-dark`  | 🖤 暗黑潮酷 | 深色背景 + 霓虹描边 | 吐槽、摸鱼、抓狂 |
| `theme-retro` | 📒 暖黄复古 | 米黄纸质 + 棕褐地面 | 亲情、回忆、叙事 |

各风格只改背景 / 地面 / 气泡 / 描边配色，角色结构和排版完全复用。

### 第 4 步：HTML 排版渲染

把每个回合的「角色 + 气泡台词」填进 `template.html` 的 `.grid`，奇偶格左右交替（`.l` / `.r`、`role` / `role right`），然后渲染：

```bash
export PATH="$PATH:/Users/a0000/Library/Python/3.9/bin"
shot-scraper comic.html --width 1080 --height 1350 --retina --wait 1500 -o comic.png
```

- 尺寸：竖版 **1080×1350**（小红书）；方版用 `--height 1080`；
- 6/9 格时 `.grid` 的 `grid-template-rows` 改 `repeat(3,1fr)`，`.grid` 高度相应调整；
- `--retina --wait 1500` 输出 2× 高清，留足字体加载时间。

### 第 5 步：汇报

用 `Read` 预览成图，给出 PNG 路径。**如实说明**：角色为 CSS 矢量（非 AI 绘图）；台词若有合并 / 改写要讲；截图含真实身份信息时已用角色昵称替代。

## 注意事项

1. **只取聊天文本**：语音时长、时间戳、"已读/撤回/拍一拍/通话邀请"等与聊天内容无关的元信息**全部忽略**，不进漫画、也不补成台词。详见第 1 步提取要点。
2. **隐私**：截图含真实姓名 / 手机号 / 头像时，漫画里一律用角色昵称代替，不出现真实身份信息；含敏感内容则提醒用户。
3. **HTML 转义**：台词里的 `< > & "` 必须转义（`&lt; &gt; &amp; &quot;`）。
4. **角色一致性**：同一个人全程同一只动物、同一配色，只换表情，别中途变种。
5. **台词精炼**：气泡放浓缩后的关键句，长句拆短或加 `<br>`，太长撑破气泡。
6. **窗格克制**：4/6/9 格，挑高光回合，别把流水账全塞进去。
7. **左右交替**：相邻格角色 / 气泡左右交替，读起来才像一来一回。
8. **风格不跑偏**：无论换什么动物 / 风格，死守「粗黑描边 + 扁平填色 + 圆润可爱 + 涂鸦感」。
9. **AI 绘图为可选项**：默认不用；用户自备文生图 key 时才升级为精细手绘立绘。
